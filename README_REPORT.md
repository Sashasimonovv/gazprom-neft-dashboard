# Отчёт о выполнении задания (дешборд курсов валют)

Ниже — кратко и по шагам, **что** сделано и **как** устроено решение. Технические команды запуска и формат API описаны в основном [`README.md`](./README.md).

---

## 1. Цель

Собрать одностраничный дешборд по макету (Figma): динамика курсов **USD, EUR, CNY** к рублю, график на **ECharts**, интерфейс на **Consta UI Kit**, данные из заданного набора с возможностью подключить **внешний API**.

---

## 2. Что реализовано

| Область | Решение |
|--------|---------|
| **Стек** | Create React App, **TypeScript**, функциональные компоненты и хуки |
| **Вёрстка и UI** | Компоненты **Consta** (`Theme`, `Card`, `Text`, `ChoiceGroup`, `Loader`, `Informer`, `Button`) и свой **CSS** для сетки карточки и боковой панели |
| **График** | **Apache ECharts**: линия по выбранной валюте, ось дат, подписи в читаемом виде, кастомный HTML-тултип по ТЗ |
| **Переключение валют** | `ChoiceGroup`: три варианта ($, €, ¥); при смене пересчитываются среднее и опции графика |
| **Среднее за период** | Считается по всем точкам ряда для активной валюты; выводится справа; на графике — **линия среднего** (`markLine`) в том же оранжевом цвете, что и серия |
| **Данные** | По умолчанию — массив из `src/data/data.ts`. Если задан `REACT_APP_RATES_API_URL` — **fetch** к API, парсинг JSON, нормализация к единому типу |
| **Состояния** | Загрузка (`Loader`), ошибка сети/формата (`Informer` + кнопка «Повторить запрос»), подсказка «Данные с API» при успешной загрузке с URL |
| **Форматирование** | **Prettier** (`.prettierrc`), числа и даты в тултипе/подписях через `Intl` (`ru-RU`) |

## 3. Как устроен код (структура и логика)

1. **`src/types/rates.ts`**  
   Задал единые типы: `CurrencyCode` (литеральный union) и `RatePoint` (`date` + `rates` по трём валютам). Все слои (данные, API, график, страница) опираются на эти типы.

2. **`src/data/data.ts`**  
   Статические ряды курсов — источник по умолчанию, без сервера.

3. **`src/api/fetchRates.ts`**  
   - Нет переменной окружения → возвращаются отсортированные по дате данные из `data.ts`, флаг `fromApi: false`.  
   - Есть URL → `GET`, разбор `unknown` как JSON, проверка массива; **`normalizeApiRow`** приводит разные форматы ответа (вложенный `rates` или плоские `usd`/`eur`/`cny` и варианты регистра) к **`RatePoint`**. При несоответствии — понятная ошибка.

4. **`src/Echarts/ReactECharts.tsx`**  
   Обёртка над ECharts: создание/обновление/resize/dispose экземпляра в жизненном цикле React, чтобы график не «течь» при перерисовках.

5. **`src/features/dashboard/chartOption.ts`**  
   Вынесена **чистая функция** `buildLineChartOption`: по точкам, валюте и уже посчитанному среднему собирается объект **`EChartsOption`** (сетка, оси, серия, тултип, `markLine`). Отдельно **`computeAverage`** — переиспользуемый расчёт для сайдбара и линии на графике. Тултип типизирован локальным минимальным типом под параметр formatter (без зависимости от внутренних типов ECharts, которые в версии библиотеки могли не экспортироваться).

6. **`src/features/dashboard/DashboardPage.tsx`**  
   Страница дешборда: состояние данных, валюты, загрузки и ошибки; `useMemo` для среднего и опций графика; `useCallback` + `useEffect` для однократной/повторной загрузки; разметка по макету (шапка, переключатель, график, блок среднего).

7. **`src/App.tsx` / `src/index.tsx`**  
   Подключение темы Consta и корневого компонента приложения.

Такое разделение упрощает проверку: типы и данные отделены от HTTP, конфиг графика — от JSX страницы.

## 4. Комментарии в коде

В **`fetchRates.ts`**, **`ReactECharts.tsx`**, **`chartOption.ts`** и в начале **`DashboardPage.tsx`** стоят **краткие комментарии**: нормализация разных форматов API, жизненный цикл экземпляра ECharts и `ResizeObserver`, тип параметра тултипа, роль основных состояний на странице. Остальной код опирается на говорящие имена функций и типы TypeScript.
