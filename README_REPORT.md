# Дешборд курсов валют — описание реализации

Кратко, что сделано в проекте и как устроен код. Запуск, переменные окружения и формат ответа API — в [`README.md`](./README.md).

---

## 1. Назначение

Одностраничный дешборд по макету [Figma](https://www.figma.com/file/CppcOcor3NP1BfrppRgd4a/Test?node-id=0%3A1&mode=dev): динамика курсов **USD**, **EUR**, **CNY** к рублю; график **Apache ECharts**; интерфейс **Consta UI Kit**; данные из локального массива или из HTTP API по адресу из переменной окружения.

---

## 2. Функциональность

| Область | Содержание |
|--------|------------|
| Стек | Create React App, TypeScript, функциональные компоненты и хуки |
| UI | Consta: `Theme`, `Card`, `Text`, `ChoiceGroup`, `Loader`, `Informer`, `Button`; отдельный CSS для карточки и боковой колонки |
| График | Линия по выбранной валюте, ось дат с подписями, кастомный HTML-тултип при наведении |
| Валюты | Переключатель на три валюты; при смене пересчитываются среднее за период и конфигурация графика |
| Среднее | Арифметическое среднее по всем дням для активной валюты; отображение справа; на графике горизонтальная линия (`markLine`) |
| Данные | Без `REACT_APP_RATES_API_URL` — сортированные ряды из `src/data/data.ts`; с URL — `fetch`, разбор JSON, приведение к единому виду |
| Ошибки и загрузка | Индикатор загрузки, сообщение об ошибке сети или неверного формата, кнопка повтора запроса; при ответе с API — текст «Данные с API» |
| Стиль кода | Prettier; форматирование чисел и дат через `Intl` (`ru-RU`) |

## 3. Структура кода

1. **`src/types/rates.ts`** — типы `CurrencyCode` и `RatePoint`: одна точка ряда — дата и три курса.

2. **`src/data/data.ts`** — статические курсы для локального режима.

3. **`src/api/fetchRates.ts`** — если URL не задан, возвращаются данные из `data.ts` и `fromApi: false`. Иначе — запрос `GET`, разбор тела как `unknown`, проверка массива; функция **`normalizeApiRow`** поддерживает вложенный объект `rates` и плоские поля `usd`/`eur`/`cny` (разный регистр ключей). Результат сортируется по полю `date`.

4. **`src/Echarts/ReactECharts.tsx`** — обёртка: создание экземпляра ECharts, `setOption` при смене пропса `option`, `ResizeObserver` для `resize`, `dispose` при размонтировании.

5. **`src/features/dashboard/chartOption.ts`** — **`buildLineChartOption`** собирает `EChartsOption` (сетка, оси, серия, тултип, `markLine`); **`computeAverage`** считает среднее для сайдбара и линии на графике. Для параметра formatter тултипа используется локальный узкий тип, чтобы не зависеть от экспорта внутренних типов из пакета `echarts`.

6. **`src/features/dashboard/DashboardPage.tsx`** — состояние рядов, валюты, загрузки и ошибки; `useMemo` для среднего и опций графика; загрузка через `useCallback` и `useEffect`; разметка шапки, переключателя, графика и блока среднего.

7. **`src/App.tsx`** — обёртка `Theme` (preset Consta) и контейнер с `DashboardPage`.

8. **`src/index.tsx`** — точка входа React (рендер корня).

Типы и статические данные отделены от HTTP; конфигурация графика вынесена из JSX страницы.

---

## 4. Комментарии в коде

Краткие пояснения стоят у нормализации строки ответа API (`fetchRates.ts`), у эффектов в обёртке ECharts (`ReactECharts.tsx`), у типа параметра тултипа (`chartOption.ts`), у типа элемента переключателя валют (`DashboardPage.tsx`). Остальное по смыслу читается по именам и типам TypeScript.
